## Quick Start (Do This Now!)

### Step 1: Search (30 minutes)
1. Open **GOOGLE_SCHOLAR_QUERIES.md**
2. Go to https://scholar.google.com
3. Copy-paste these 4 queries (in order):
   ```
   "Computer Networks" "reinforcement learning" "task placement" OR "service placement" 2023 OR 2024
   
   "Computer Networks" "federated learning" "edge computing" 2023
   
   "Computer Networks" "graph neural network" "edge computing" 2023 OR 2024
   
   "Computer Networks" "semantic" "edge computing" OR "edge orchestration" 2023
   ```
4. Browse first 10-15 results from each query
5. Save PDFs of 8-10 promising papers

### Step 2: Filter (30 minutes)
1. Open **PAPER_DATA_COLLECTION.md**
2. Skim abstracts and results sections
3. Select 3-5 best papers based on:
   - Uses similar metrics (latency, power, etc.)
   - Published 2023-2025
   - Reports numerical results
   - Addresses similar problem

### Step 3: Extract Data (2-3 hours)
1. For each selected paper, find their results table
2. Extract key numbers (latency, power, communication, etc.)
3. Fill in **PAPER_DATA_COLLECTION.md** template
4. Note table/figure references

- Filled PAPER_DATA_COLLECTION.md

## Important Notes

### ✅ DO:
- Extract real numbers from published papers
- Cite sources for all data
- Note differences in experimental setup
- Be honest about comparison limitations
- Choose papers from Computer Networks journal (primary) or top-tier venues

### ❌ DON'T:
- Make up numbers
- Cherry-pick only favorable comparisons
- Claim unfair advantages
- Ignore setup differences
- Compare apples to oranges without noting it

## Alternative If Short on Time

If you can't find enough Computer Networks papers, you can:

1. **Keep your 4 implemented baselines** (you have these)
2. **Add 2-3 simple baselines:**
   - Round-Robin (10 lines of code)
   - Greedy-Power (10 lines of code)
   - Greedy-Latency (10 lines of code)

3. **Add 1-2 papers from related venues:**
   - IEEE TMC (Transactions on Mobile Computing)
   - ACM TOCS (Transactions on Computer Systems)
   - IEEE TPDS (Transactions on Parallel and Distributed Systems)

4. **Write clear methodology:**
   "We compare against implemented baselines (FlatFedPPO, HierFedPPO, HSQF, Random, 
   Round-Robin, Greedy) and position our work relative to recent literature [citations]."

This is still acceptable but slightly weaker than direct comparison.
